# מערכת רב-לשוניות (i18n) — בשבילנו

מסמך תחזוקה קצר למנגנון השפות שנוסף לאתר (שלב א.1 בתוכנית השדרוג).

## איך זה עובד

האתר נשאר **סטטי טהור**. `build.py` מייצר עמוד נפרד לכל שפה מתוך תבנית אחת + מילוני תרגום:

| שפה | כיוון | כתובת |
|-----|-------|-------|
| עברית (ברירת מחדל) | RTL | `/` (שורש) |
| אנגלית 🇺🇸 | LTR | `/en/` |
| צרפתית 🇫🇷 | LTR | `/fr/` |
| ספרדית 🇪🇸 | LTR | `/es/` |

- **מילוני תרגום**: `i18n/he.json`, `en.json`, `fr.json`, `es.json` — JSON מקונן, מפתחות בנקודות (`hero.title`). הערכים יכולים להכיל HTML (למשל `<span class="c-green">…`).
- **תבניות**: `templates/*.html` עם טוקנים `{{key}}`. כרגע: `index.html` בלבד (פיילוט).
- **partials משותפים**: `partials/nav.html` + `partials/footer.html` — מתורגמים אף הם דרך טוקנים, כולל מתג השפה עם דגלי SVG.
- **מתג שפה**: `assets/main.js` (בלוק "Language switcher") מחשב בזמן ריצה את הקישור הנכון לכל שפה עבור העמוד הנוכחי, מסמן את הפעילה, וזוכר את הבחירה ב-`localStorage`. ההחלפה מנווטת באותה כרטיסייה (לא נפתחת חדשה).
- **דגלים**: SVG inline (sprite) — רנדור עקבי בכל מערכת הפעלה, משקל אפסי.

## הרצה

```bash
python3 build.py
```

הסקריפט אידמפוטנטי. הוא:
1. מייצר את כל גרסאות השפה של העמודים שב-`TEMPLATES`.
2. מרענן את ה-nav/footer (כולל מתג השפה) בעמודי העברית ה-legacy שעדיין לא תורגמו.

## הוספת עמוד מתורגם חדש (לדוגמה `about`)

1. צרו `templates/about.html` מתוך `about.html` הקיים: החליפו טקסטים בטוקנים `{{…}}`, נתיבי נכסים ל-`/assets/...`, הוסיפו בלוק hreflang ומרקרים `<!-- @nav -->` / `<!-- @footer -->`.
2. הוסיפו את המפתחות לכל 4 קבצי ה-`i18n/*.json`.
3. הוסיפו `"about"` לרשימת `TEMPLATES` שב-`build.py`, ולמפה `TRANSLATED` שב-`assets/main.js`.
4. הריצו `python3 build.py`.

## הוספת שפה חדשה

1. צרו `i18n/<lang>.json` (העתיקו מ-`en.json` ותרגמו), כולל `lang_menu.current_flag` / `current_code`.
2. הוסיפו את הקוד ל-`LANGS` ב-`build.py` ול-`LANGS`/`TRANSLATED` ב-`main.js`.
3. הוסיפו דגל (symbol SVG) ב-`partials/nav.html` ופריט במתג השפה (dropdown + drawer).
4. עדכנו את בלוק ה-hreflang בתבניות ואת `sitemap.xml`.
